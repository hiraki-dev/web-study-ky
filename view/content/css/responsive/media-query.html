メディアクエリーとは

モバイル、タブレット、PCなどのデバイスによって画面の表示サイズは変化する。
それに対応すべく、ビューポートの表示領域に応じてスタイルを適応できるCSSの機能がメディアクエリーである。
メディアクエリーを用いることで端末別にデザインを適用できる。

ビューポートメタタグ

メディアクエリーを使用する際はHTML内にビューポートメタタグを定義しておくこと。
指定しておかないと表示が崩れるなどの弊害が起きる。 

<meta name="viewport" content="width=device-width,initial-scale=1" />

メディアクエリーの構文

メディアクエリーは@mediaルールを指定して定義する。
以下はメディアクエリーの構文である。 

@media メディアタイプ 論理演算子 (メディア特性) {
/* 条件に一致した際に適用するCSSを記述 */
  div {
    ...
  }
}
  • メディアタイプではデバイス画面、印刷ページ、音声出力装置などのメディアを指定する。
  • メディア特性では幅や高さなどのメディアの領域を指定する。
  • and条件、or条件、not条件などの論理演算子を指定できる。

メディアタイプ

適用したいメディアに応じて以下のタイプを指定できる。
以下に記載していないメディアタイプは非推奨のため省略する。

メディアタイプ 説明
all 全メディアタイプ
screen デバイス画面
print 印刷プレビュー
speech 音声出力デバイス

以下は記述例。

/* 画面 */
@media screen and (...) {
  ...
}

/* 印刷プレビュー */
@media print and (...) {
  ...
}

/* 全デバイス */
@media all and (...) {
  ...
}

/* 全デバイスの場合、以下のように省略できる */
@media (...) {
  ...
}

メディア特性

画面の幅や高さなどのメディアの領域を指定する。
以下に記載していないメディア特性は非推奨のため省略する。

メディア特性 説明
width 幅
max-width 幅（最大値）
min-width 幅（最小値）
height 高さ
orientation 縦長モード (portrait) / 横長モード (landscape)
aspect-ratio 幅と高さの比率(アスペクト比)
resolution 解像度
color 色
grid グリッド

以下は記述例。

/* 画面幅が1024px */
@media screen and (width: 1024px) {
  ...
}

/* 画面幅が768px以上 */
@media screen and (max-width: 768px) {
  ...
}

/* 画面が横長モード */
@media screen and (orientation: landscape) {
  ...
}

比較演算子を用いた記述

widthと比較演算子を用いて簡略的に記述できる。

@media screen and (min-width: 1024px) {

}

/* 上記と同等 */
@media screen and (width < 1024px) {

}

/* 上記と違い、1024pxも含む */
@media screen and (width =< 1024px) {

}

範囲指定

画面幅の範囲などを指定する場合、以下のように省略できる。

/* 以下は同等 */
@media screen and (min-width: 300px) and (max-width: 500px) {
  ...
}

@media screen and (300px <= width <= 500px) {
  ...
}

論理演算子

メディアクエリーは論理演算子を用いた複雑な条件を指定できる。

/* and条件の例 */
@media screen and (min-width: 1024px) {
  /* 画面の幅が1024px以下の場合に適用 */
}

/* or条件の例（カンマで区切る） */
@media screen and (min-width: 768px) , screen and (orientation: landscape) {
  /* 画面の幅が768px以下の場合、または横長モードの場合に適用 */
}

/* not条件の例 */
@media not screen and (min-width: 1024p000x) {
  /* 画面の幅が768px以下の場合は適用しない */
}

メディアクエリーのネスト

メディアクエリーはネストして指定できる。
ネストして指定されたメディアクエリーは、親クエリにand条件として追加される。 

@media screen and (width > 768px) {
  div {
    ...
  }

  @media screen and (orientation: landscape) {
    section {
      ...
    }
  }
}

/* 上記と同等 */
@media screen and (width > 768px) {
  div {
    ...
  }
}

@media screen and (width > 768px) and (orientation: landscape) {
  section {
    ...
  }
}

モバイルファースト/ブレークポイント

GoogleはWebページのインデックスやランク付けの指標として、モバイル端末の表示を優先している。 これをモバイルファーストといい、レイアウトデザインはモバイルから行うのが望ましいとされている。（対義語がデスクトップファースト） 

/* モバイル */
selector {
  ...
}

/* タブレット */
@media screen and (min-width: 768px) {
  ...
}
            
/* デスクトップ */
@media screen and (min-width: 1348px) {
  ...
}

また、メディアクエリーを適用する位置のことをブレークポイントという。
ブレークポイントはデバイスに合わせて置くのが基本だが、デバイスが多様化している現状、デバイスに合わせて設定するよりもコンテンツの表示限界に合わせて設置することが望ましい。
